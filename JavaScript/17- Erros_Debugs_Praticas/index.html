Erros:

Ao executar o código JavaScript, diferentes erros podem ocorrer. Os erros podem ser erros de codificação cometidos pelo programador, erros devidos a entrada incorreta e outras coisas imprevisíveis.

try, catch:

A instrução try permite definir um bloco de código a ser testado quanto a erros enquanto está sendo executado.

A instrução catch permite definir um bloco de código a ser executado, caso ocorra um erro no bloco try.

Sintaxe: try { Bloco de código para try } catch(err) { Bloco de código para lidar com erros }

        
            try {
                adddlert("Welcome guest!"); //O JavaScript detecta addledrt como um erro e executa o código catch para tratá-lo.
            }
            catch(err) {
               console.log(err.message);
            }
            >>Resultado: adddlert is not defined

throw:

A instrução throw permite que você crie um erro personalizado.

finally:

A instrução finally permite que você execute o código, após try e catch, independentemente do resultado.

Sintaxe: try { Bloco de código para try } catch(err) { Bloco de código para lidar com erros } finally { Bloco de código a ser executado independentemente do resultado try/catch }

Objeto de Erro:

EvalError:

Um EvalError(erro de avaliação) indica um erro na função eval().

RangeError:

Um RangeError é lançado se você usar um número que está fora do intervalo de valores legais.

        
            EX: let num = 1;
                try {
                    num.toPrecision(500);
                }
                catch(err) {
                    console.log(err.name); >>Resultado RangeError
                }

ReferenceErro:

Um ReferenceError é lançado se você usar (referenciar) uma variável que não foi declarada.

        
            EX: let x = 5;
                try {
                    x = y + 1;
                }
                catch(err) {
                    console.log(err.name); >>Resultado: ReferenceError
                }

SyntaxError:

Um SyntaxError é lançado se você tentar avaliar o código com um erro de sintaxe.

        
            EX: try {
                    eval("alert('Hello)");
                }
                catch(err) {
                    console.log(err.name); >>Resultado: SyntaxError
                }

TypeError:

Um TypeError é lançado se você usar um valor que está fora do intervalo de tipos esperados.

        
            EX: let num1 = 1;
                try {
                    num.toUpperCase();
                }
                catch(err) {
                    console.log(err.name);
                }

URIError:

Um URIError é lançado se você usar caracteres ilegais em uma função URI.

        
            EX: try {
                    decodeURI("%%%");
                }
                catch(err) {
                    console.log(err.name);
                }

Depuração:

Procurar (e corrigir) erros no código de programação é chamado de depuração de código.

Normalmente você ativa a depuração em seu navegador com a tecla F12 e seleciona "Console" no menu do depurador.

Se o seu navegador suportar depuração, você pode usar console.log() para exibir valores JavaScript na janela do depurador.

Guia de Estilos:

Nomes de variáveis:

No W3schools usamos camelCase para nomes de identificadores (variáveis e funções)

        
            EX: primeirotNome = "João";
                segundoNome = "Gomes";
            
                preco = 19.90;
                taxa = 0.20;
            
                precoCompleto = preco + (preco * taxa);

Operadores:

Sempre coloque espaços ao redor dos operadores ( = + - * / ) e depois de vírgulas.

        
            EX: let x = y + z;
                const meuArray = ["Volvo", "Saab", "Fiat"];

Recuo de código:

Sempre use 2 espaços para indentação de blocos de código

        
            EX: function toCelsius(fahrenheit) {
                    return (5 / 9) * (fahrenheit - 32);
                }

Regras de Declaração:

Regras gerais para declarações simples: Sempre termine uma declaração simples com um ponto e vírgula.

        
            EX: const cars = ["Volvo", "Saab", "Fiat"];

Regras gerais para declarações complexas (compostas): Coloque o colchete de abertura no final da primeira linha; Use um espaço antes do colchete de abertura; Coloque o colchete de fechamento em uma nova linha, sem espaços iniciais; Não termine uma instrução complexa com um ponto e vírgula.

        
            EX: function toCelsius(fahrenheit) {
                    return (5 / 9) * (fahrenheit - 32);
                }

Comprimento da linha < 80:

Para facilitar a leitura, evite linhas com mais de 80 caracteres. Se uma instrução JavaScript não couber em uma linha, o melhor lugar para quebrá-la é após um operador ou uma vírgula.
